* Overview

This repository hosts a collaborative engineering project implementing
*Diceps* (Distributed Ising Computing on ESP Platform System) across a
physical cluster of ESP32-C6 DevBoards.

An Ising machine is a specialized computational framework utilizing the computational properties of networks of interacting spins that naturally evolve toward a minimum energy state. This project moves away from purely algorithmic software simulations, executing the dynamical equations directly on a distributed, multi-MCU hardware platform.

** Core Objectives
- Explore, implement, and benchmark various interconnect modes (e.g., high-speed SPI, CAN bus/TWAI, Wi-Fi, 802.15.4) to determine the ideal topology for low-latency spin-state broadcasting.
- Develop deterministic, low-overhead communication protocols optimized for the ESP32-C6 RISC-V architecture.
- Validate physical cluster performance against the state-of-the-art mathematical models of the non-binary dynamical Ising machines developed at the University of Michigan.

* Documentation & Workflow Philosophy

This project prioritizes explicit knowledge propagation alongside functional firmware implementation. 

- *Primary Format*: The master document format for this repository is Org-mode (=.org=). 
- *GitHub-First View*: Contributors are not expected to be power users of Emacs or Org-mode. All documents are written using a basic, clean subset of syntax to ensure they render perfectly inside the standard GitHub web interface.
- *Task Authority*: The GitHub Issue Tracker is the absolute source of truth for task tracking and holds complete precedence over files in the repository. Localized =to_add.lst= task files and inline =TODO= markers serve purely as suggestions and roadmap guides.

For an in-depth breakdown of developer workflows, coding conventions, and documentation standards, please read the *Contributor Onboarding Guide* located at:
